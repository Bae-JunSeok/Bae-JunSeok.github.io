Project 01

TaleMory

가족의 여행 사진과 목소리를 바탕으로 영어 동화책, 한국어 번역, 가족 목소리 음성을 자동 생성하는 AI 동화책 서비스입니다.

TTS 성능 개선 6인 팀 AI Pipeline Backend 2026.04 - 2026.05

프로젝트 개요

개발 배경
여행 사진과 가족 목소리를 동화책으로 보존하고, 자녀 영어 학습까지 연결하는 AI 서비스 구현
주요 기능
사진 업로드, 줄거리/스토리보드 생성, 그림 스타일 선택, voice clone, 최종 미리보기, 발행
팀 구성
6인 팀 · AI TTS 파이프라인 및 백엔드/프론트 일부 보완 담당
기술 스택
Kotlin, Spring Boot, FastAPI, RabbitMQ, Qwen TTS, OpenAI, Gemini, Replicate, MySQL, Redis, S3, Docker

핵심 기여

  • FastAPI TTS route, schema, service, 테스트를 구성해 음성 합성 기본 흐름 구현
  • Preview TTS와 본문 TTS를 RabbitMQ 작업으로 분리하고 요청/결과 메시지 스키마 정리
  • 생성된 음성 파일을 S3에 저장하고 백엔드가 key와 URL을 함께 다룰 수 있도록 응답 DTO 보완
  • Qwen TTS 서버 클라이언트와 별도 FastAPI 서버 코드를 추가해 외부 GPU 서버 연동 흐름 구성
  • TensorRT 적용으로 평균 합성 시간 14.00초에서 6.97초, Story TTS 전체 처리 8분 45초에서 4분 04초로 개선
문제 해결

긴 AI 음성 생성 작업을 사용자 흐름에서 분리

동화 본문 전체 TTS는 요청 시간이 길고 실패 복구가 필요했습니다. 동기 요청으로 처리하면 사용자는 응답을 기다려야 하고, 서버는 무거운 AI 작업에 묶이게 됩니다.

비동기화

RabbitMQ topic exchange 기반으로 AI 요청과 결과를 분리하고, consumer가 작업 상태를 갱신하도록 구성했습니다.

저장 흐름

합성 오디오는 S3에 저장하고, 백엔드는 S3 key와 접근 URL을 응답에 포함해 프론트 재생 흐름과 연결했습니다.

성능 개선

TensorRT 적용 전후 로그를 비교해 평균 합성 시간과 weighted RTF를 절반 수준으로 줄였습니다.

Before

TTS 생성이 길어질수록 요청 흐름이 무거워지고, 실패 지점과 재처리 기준을 추적하기 어려웠습니다.

After

Preview TTS와 Story TTS를 큐 기반 작업으로 분리하고, 결과 메시지를 통해 백엔드 상태와 S3 저장 결과를 갱신했습니다.

성과

평균 합성 시간 14.00초에서 6.97초, 34문장 Story TTS 전체 처리 8분 45초에서 4분 04초로 개선했습니다.

조회 구조 개선 6인 팀 Recommendation Backend 2026.02 - 2026.03

프로젝트 개요

개발 배경
여행지 선택 시 흩어진 취향, 물가, 항공, 관광지 정보를 한 흐름에서 비교하기 어려운 문제 해결
주요 기능
Google OAuth, YouTube 관심사 분석, AI 도시 추천, 도시 상세, 생활물가 비교, 항공권 트렌드, 북마크
팀 구성
6인 팀 · 추천/도시 API, 점수 산정 로직, 성능 개선 담당
기술 스택
Java 17, Spring Boot, JPA, Spring Security, Spring AI, MySQL, MongoDB, Redis, Docker, Nginx

핵심 기여

  • 추천/비추천 도시 상세 응답 DTO와 도시 조회 API 확장

  • 태그 점수 비중을 조정하고 위험도/예산 태그를 추천 점수에 반영하도록 개선
  • 도시별 항공권, 생활비, 위험도 반복 조회를 배치 조회와 Map 조립 구조로 변경
  • 추천 상세 API에 단계별 시간 로그를 추가해 9.5초 중 뉴스 4.9초, 추천 이유 생성 2.7초 병목 확인
조회 구조N+1 제거

80개 도시 기준 반복 조회를 항공권, 생활비, 위험도 단위 배치 조회로 정리했습니다.

병목 계측9.5초 분석

단계별 로그로 뉴스 검색과 추천 이유 생성이 주요 지연임을 확인했습니다.

추천 품질근거 강화

취향 태그와 위험도/예산 태그를 함께 반영해 추천 설명의 설득력을 높였습니다.

문제

도시 목록에서 항공권, 생활비, 위험도 정보를 도시별로 반복 조회하며 데이터 수가 늘수록 응답 시간이 불안정해졌습니다.

해결

도시 ID 목록을 기준으로 필요한 데이터를 한 번에 조회하고, Map으로 조립해 반복 쿼리 수를 줄였습니다.

추가 판단

추천 상세 API에는 단계별 실행 시간 로그를 넣어 뉴스 검색과 추천 이유 생성 병목을 분리해서 확인했습니다.
